Firestore（NoSQL）をベースとしたデータ定義書を作成します。 「学習・サッカー・生活」の3観点を軸に、前週の「Will（意志）」を翌週の「Goal（予定）」へと昇華させる**スパイラルアップ構造**をデータの持たせ方で実現します。

# **データ定義書：Spiral Up Logger**

## **1\. データベース構造の概要**

Firebase Cloud Firestoreを採用し、ユーザーごとの成長を時系列で追える構造にします。

* **Users (Collection)**: ユーザー基本情報  
* **WeeklyReflections (Collection)**: 週次の振り返りデータ（メイン）  
* **DailyRoutines (Collection)**: 毎日の習慣トラッカー  
* **Goals (Collection)**: 中長期目標（マンダラチャートの要素）

## **2\. コレクション詳細**

### **① WeeklyReflections (週次振り返り)**

1on1の核心となるデータです。土曜朝の入力画面はこのデータを生成・更新します。

| フィールド名 | 型 | 説明 |
| :---- | :---- | :---- |
| id | String | ドキュメントID (例: 2026\_W11) |
| userId | Reference | ユーザーへの参照 |
| weekStartDate | Timestamp | その週の月曜日の日付 |
| status | Enum | DRAFT (入力中) / COMPLETED (1on1完了) |
| **study** | Map | **学習カテゴリ** |
| ├ plan | String | 前週の nextWill から自動コピーされる予定 |
| ├ actual | String | 実施した具体的な内容（ワークの進捗等） |
| ├ score | Number | 達成度 (0-100) |
| ├ good | String | 良かった点・再現したい工夫 |
| ├ bad | String | 障害・改善が必要な点 |
| ├ nextWill | String | **次週への具体的な行動予定** |
| **soccer** | Map | **サッカーカテゴリ** |
| ├ plan / actual / score | (同上) | 目標（Aクラス入り等）に対する進捗 |
| ├ good / bad / nextWill | (同上) | 練習・試合の質、アジリティ等の手応え |
| **life** | Map | **生活カテゴリ** |
| ├ plan / actual / score | (同上) | ルーティン、睡眠、栄養（マジ・テin等） |
| ├ good / bad / nextWill | (同上) | メンタル、家族・友人との距離感 |
| inventionNote | String | 今週編み出した「新しい自分独自の工夫」 |
| mentorComment | String | 1on1時に父（メンター）から受けたアドバイス |

### **② DailyRoutines (デイリーログ)**

土曜日の振り返りを楽にするための、日々のクイック記録です。

| フィールド名 | 型 | 説明 |
| :---- | :---- | :---- |
| date | String | 日付 (YYYY-MM-DD) |
| items | Array\<Map\> | ルーティン項目の成否 |
| └ taskName | String | 「朝の3ステップ（座・屈・跳）」「マジ・テin摂取」等 |
| └ completed | Boolean | 実施の有無 |
| sleepHours | Number | 睡眠時間 (7.5〜10時間の計測用) |

## **3\. スパイラルアップの実装ロジック**

このアプリの最大の特徴である「前週からの引き継ぎ」を、プログラム側で以下のように制御します。

1. **新規作成時（土曜朝 07:00）**:  
   * WeeklyReflections から weekStartDate が最新のドキュメント（先週分）を取得。  
   * 先週の nextWill の値を、今週の plan フィールドの初期値としてセット。  
2. **入力時**:  
   * 「予定（先週の自分が決めたこと）」が画面に表示されている状態で、現在の「結果」を隣に入力するUIを提供。  
3. **1on1完了時**:  
   * 確定した nextWill を保存。これが翌週のトリガーとなる。

## **4\. UIコンポーネントへの紐付け**

* **学習セクション**: study.plan に基づき、「ワークのページ数」や「理解度」を入力するフォームを表示。  
* **サッカーセクション**: soccer.score をスライダーで入力し、soccer.nextWill には「来週強化するスキル（例：裏への対応）」を記載。  
* **生活セクション**: DailyRoutines の1週間分の集計をグラフ表示し、それを見て life.good/bad を記入。
